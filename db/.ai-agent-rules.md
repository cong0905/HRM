# 🤖 AI Agent Guidelines & Project Context

Tài liệu này giúp các AI Agent hiểu bối cảnh và quy tắc phát triển của dự án HRM để tránh viết code sai cấu trúc.

## 1. Bối cảnh dự án (Project Context)
- **Tên dự án:** Hệ thống Quản lý Nhân sự (HRM System).
- **Công nghệ:** .NET 8.0, WinForms.
- **Cơ sở dữ liệu:** SQL Server thông qua Entity Framework Core (Code-First).
- **Kiến trúc:** N-Tier (5 tầng) để đảm bảo tách biệt trách nhiệm.

## 2. Cấu trúc 5 Tầng (Layered Architecture)
Mọi AI Agent phải tuân thủ nghiêm ngặt luồng dữ liệu: `GUI -> BLL -> DAL -> Database`.

1.  **HRM.Domain (Entities):**
    - Chứa các class thực thể (Database Tables).
    - **Quy tắc:** Chỉ chứa properties, không chứa logic xử lý hay dependencies.
2.  **HRM.DAL (Data Access Layer):**
    - Chứa `HrmDbContext`, `Migrations`, `Repositories`.
    - Cấu hình Database phải thực hiện trong thư mục `Configurations/` bằng **Fluent API**.
    - **Quy tắc:** Không sử dụng DataAnnotations (Attributes) cho các ràng buộc phức tạp.
3.  **HRM.BLL (Business Logic Layer):**
    - Chứa các `Services` xử lý nghiệp vụ.
    - **Quy tắc:** Là nơi duy nhất được phép xử lý logic. Mọi yêu cầu từ GUI phải qua Service.
4.  **HRM.GUI (Presentation Layer):**
    - Chứa WinForms (`System.Windows.Forms`).
    - **Quy tắc CẤM:** Không được gọi trực tiếp `DbContext` hoặc `Repository` từ GUI. Phải thông qua Service tại BLL.
5.  **HRM.Common (Shared):**
    - Chứa DTOs (Data Transfer Objects), Helpers, Constants.

## 3. Quy tắc lập trình cho AI (Strict Rules)

- **Database First? NO:** Dự án dùng **Code-First**. Mọi thay đổi DB phải qua `Add-Migration`.
- **Primary Keys:** Luôn đặt tên theo định dạng `Ma + [Tên Thực Thể]` (VD: `MaNhanVien`, `MaTaiKhoan`).
- **Seeding Data:** Dữ liệu khởi tạo (như Admin, danh mục) phải nằm trong các file `Configuration` của DAL.
- **Authentication:** 
    - Sử dụng thư viện `BCrypt.Net-Next`.
    - Mật khẩu admin mặc định: `admin123`.
    - Hash chuẩn: `$2a$11$KWj.ShrxV4ZnlRTKcJZmWODKnW5JlDBA.n3P2pu68ifqc06xQcCve`.
- **Tự động hóa:** `HRM.GUI/Program.cs` đã có lệnh `context.Database.Migrate()`. Đừng xóa nó, nó giúp DB tự tạo khi chạy lần đầu.
- **Dọn dẹp:** Không tạo các Project con để test (như `TestAuth`) bên trong thư mục `src`. Dùng Unit Test project nếu cần.

## 4. Quy ước đặt tên (Naming Conventions)
- Interface: Bắt đầu bằng chữ `I` (VD: `ITaiKhoanService`).
- Repository: Kết thúc bằng `Repository` (VD: `NhanVienRepository`).
- Service: Kết thúc bằng `Service` (VD: `AuthService`).
- Cấu hình EF: Kết thúc bằng `Configuration` (VD: `ChamCongConfiguration`).

*Lưu ý: Nếu bạn là một AI Agent, hãy đọc kỹ cấu trúc hiện có trước khi đề xuất bất kỳ thay đổi nào. Đừng "code lung tung" phá vỡ kiến trúc N-Tier.*
